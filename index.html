Загрузка спринта по ролям

План и факт рассчитываются автоматически в человеко-днях по конкретным задачам текущего спринта. Светлая линия показывает план, заполнение — выполненный объём.

Свободная ёмкость по ролям

Показывается ёмкость роли, плановая нагрузка выбранных задач и остаток. Аналитик и дизайнер могут не участвовать в определении узкого места, но их оценки сохраняются.

История планирования

История строится по сохранённым спринтам. В ячейках: план / факт / ёмкость роли в человеко-днях.

Настройки расчёта и команды период, численность, потребление задачи, ёмкость и формула

Параметры периода

Формула: ёмкость роли = (люди × рабочие дни − отсутствия) × коэффициент × (1 − резерв). Для поставки текущего спринта аналитик и дизайнер по умолчанию не считаются ограничением, потому что основную подготовку выполняют до планирования.

Формула

Предварительный факт по GitLab

Статусы сопоставляются с ролями без изменения кода. Для одного assignee используется его часовой пояс. Расчёт только предлагает значения; факт сохраняется после ручного подтверждения.

Часовые пояса исполнителей

Список формируется из assignee GitLab-задач. Смещение показывается в формате UTC±ЧЧ:ММ. Для задачи с одним исполнителем используется его часовой пояс; значение «По умолчанию» означает общий часовой пояс команды.
Исполнитель GitLab username Часовой пояс
Тип GitLab-задачи Роль факта Статусы, разделённые запятой

Сопоставление выполняется по точным названиям labels с учётом регистра. Несколько выбранных статусов одной роли считаются как объединение интервалов и не удваивают время. Пустая строка отключает соответствующее правило.
